# Phase 3: Project Design Phase

This folder details the system architecture, database schema, and problem-solution fits for the platform.

---

## 1. Problem-Solution Fit Matrix

| Customer Pain | Pain Reliever / Feature | Value Created |
| :--- | :--- | :--- |
| **High anxiety over debt payments** | *DTI & EMI Ratios on Dashboard* | Gives the borrower a clear, objective understanding of their financial load. |
| **Unaware of what settlement to ask for** | *Settlement Predictor Engine* | Suggests target percentage reductions based on delinquency and stress levels. |
| **Unable to draft professional letters** | *AI Hardship Letter Generator* | Drafts print-ready formal letters automatically using Google Gemini AI. |
| **Lacks legal negotiation strategy** | *AI Strategy Advisor* | Provides 3-4 bulleted negotiation tips specific to the creditor type and situation. |
| **Loses track of past proposals** | *Negotiation History Accordion* | Persistently saves and logs all generated strategies and letters. |

---

## 2. Solution Architecture Flow
The platform is designed using a decoupled client-server architecture:

```
[USER LAYER] 
      │ (Interacts with React Web Client)
      ▼
[PRESENTATION LAYER] ➔ React.js (Vite) Single Page App (Single-pass layout, CSS glassmorphism)
      │ (HTTPS REST API Queries / JWT tokens)
      ▼
[API ROUTER LAYER] ➔ FastAPI Backend Services (CORS validation, JWT Middleware)
      │
      ├─➔ [FINANCIAL PROCESSING ENGINE] (DTI & Settlement prediction calculations)
      │
      └─➔ [GEN-AI LAYER] ➔ Google Gemini API Integration (Hardship letter compiling)
      │
      ▼
[DATABASE PERSISTENCE LAYER] ➔ SQLAlchemy ORM ➔ SQLite (Local) / PostgreSQL (Cloud)
```

## 3. Database Schema Reference

The database consists of 6 tables linked by primary and foreign keys:

### 1. `users` (Primary Profile)
*   `UserID` (Integer, Primary Key) - Unique identifier for the user.
*   `Name` (String) - User's full name.
*   `Email` (String, Unique) - User's email (login username).
*   `Password` (String) - Hashed password string.
*   `MonthlyIncome` (Float) - User's gross monthly income.
*   `MonthlyExpenses` (Float) - User's basic living expenses.

### 2. `financial_profile` (Ratios Table)
*   `ProfileID` (Integer, Primary Key)
*   `UserID` (Integer, Foreign Key -> `users.UserID`) - 1:1 link to User.
*   `EMI_Ratio` (Float) - Calculated as `Total EMI / Income`.
*   `DTI_Ratio` (Float) - Calculated as `Total Debt / Annual Income`.
*   `MonthlySurplus` (Float) - Calculated as `Income - Expenses - EMIs`.
*   `StressLevel` (String) - Low, Medium, High, or Severe.

### 3. `loans` (Liability Table)
*   `LoanID` (Integer, Primary Key)
*   `UserID` (Integer, Foreign Key -> `users.UserID`) - 1:N link to User.
*   `LenderName` (String) - Name of the creditor.
*   `LoanType` (String) - Credit Card, Personal Loan, Mortgage, etc.
*   `OutstandingAmount` (Float) - Total balance unpaid.
*   `InterestRate` (Float) - Annual Percentage Rate (APR).
*   `EMI` (Float) - Active monthly payment.
*   `OverdueMonths` (Integer) - Months delinquent.

### 4. `settlement_prediction` (Calculations Table)
*   `SettlementID` (Integer, Primary Key)
*   `LoanID` (Integer, Foreign Key -> `loans.LoanID`) - 1:N link to Loan.
*   `SuggestedSettlement` (Float) - Recommended payoff percentage (e.g. 45.0).
*   `RiskCategory` (String) - Default litigation risk level (Low, Medium, High, Severe).
*   `PredictedAmount` (Float) - Calculated payoff value in dollars.

### 5. `ai_negotiation` (AI Output Log)
*   `AI_ID` (Integer, Primary Key)
*   `LoanID` (Integer, Foreign Key -> `loans.LoanID`)
*   `UserID` (Integer, Foreign Key -> `users.UserID`)
*   `NegotiationStrategy` (Text) - Bullets of advice generated by Gemini.
*   `NegotiationLetter` (Text) - Complete hardship letter text.
*   `GeneratedDate` (DateTime) - Date of creation.

### 6. `ai_history` (Audit Log)
*   `HistoryID` (Integer, Primary Key)
*   `UserID` (Integer, Foreign Key -> `users.UserID`)
*   `GeneratedContent` (Text) - Concise record of the event.
*   `QueryType` (String) - "Settlement Prediction Review" or "Hardship Letter Draft".
*   `Timestamp` (DateTime) - Audit date.
